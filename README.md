# cube-analyzer

## Description

群論に基づくルービックキューブの表現・可視化・自動解法プロジェクト。

### 群論的表現
状態と操作を同一の `State` クラスで表現する。キューブの状態遷移は群演算として扱い、合成・逆元・単位元・繰り返しをすべて `State` オブジェクトで統一する。

### 可視化
- Python: 展開図 (2D net tensor / ASCII)、3D表示
- Swift: SceneKit / RealityKit を用いたインタラクティブ3Dアプリ

### 自動解法モデル (2方針並行)

1. **Transformer (LLM的アプローチ)**  
   キューブ状態と操作履歴をトークン列として入力し、次の操作トークンを予測する自己回帰モデル。

2. **離散拡散モデル**  
   ランダムな操作の適用を前向き拡散（ノイズ付加）とみなし、逆拡散過程として解法を学習する D3PM 系のアプローチ。

訓練はランダムスクランブルから始め、成果が出たら CFOP 法 (OLL/PLL) に基づく操作列に限定したモデルも作成する。

### 学習・推論
MLX (Apple Silicon) を用いて学習・推論を行う。Swift アプリには MLX Swift を組み込む。

## 群演算 API

| 演算 | 意味 |
|---|---|
| `a @ b` | `a` に `b` を適用（群の合成） |
| `a @= b` | `a` をインプレースで更新 |
| `~a` | 逆元 |
| `n * a` | 操作 `a` を `n` 回繰り返し（負数は逆方向） |
| `a == b` | 状態の等価比較 |
| `State()` | 解いた状態（単位元） |

基本操作: `MOVES['U']`, `'D'`, `'L'`, `'R'`, `'F'`, `'B'`

## 開発上の注意
- 状態を変更する前に `.clone()` を使う（`apply` はミュータブル、`get_applied` / `@` はイミュータブル）
