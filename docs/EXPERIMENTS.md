# 実験記録と今後の方針

ルービックキューブ ML ソルバーの学習・評価結果、ならびに手法レビュー
（Fable 5 による批判的レビューを含む）と、それに基づく今後の改善ロードマップ。

最終更新: 2026-06（small モデルでの一連の実験完了時点）

---

## 1. セットアップ

- **モデル** `CubeSolver`（`source/model/solver.py`）: goal条件付き Transformer。
  goal 20 トークン + current 20 トークン = 40 トークン入力、18クラス（6面×{1,2,3}回転）出力。
  オプションでノイズレベル `t` を入力。
  - small: d=128 / 4層 / 4ヘッド → 812K params
  - medium: d=256 / 6層 / 8ヘッド → 4.78M params
- **学習データ**（`source/data.py`）
  - **diffusion**: solved に `t ~ U(1, t_max)` 回ランダム手を適用し、
    ターゲット = 最後に適用した手の逆手。
  - **BC（CFOP教師）**: CFOPソルバーの軌跡から (状態, 次手) を生成。goal=identity、
    t=残り手数。`--diverse-pool` で F2L ペア順・AUF をランダム化し可変深さに。
- **CFOP教師**（`source/cfop.py`）: Cross=190k状態テーブル / F2L=PDB誘導IDA* /
  OLL・PLL=最終層部分群62208状態テーブル。100%正解・平均約90手。
- **評価**（`source/infer.py`）: 学習済みモデルにスクランブルを解かせ、貪欲 or
  ビームでロールアウト。ステップ k で `t = max(1, scramble_depth - k)` を与える。
  成功 = `max_steps` 以内に solved 到達。

## 2. 結果（small 812K, beam-8, n=100）

diffusion は 80k ステップ（20k + 60k 継続）、BC-diverse は 40k ステップ。

| スクランブル深さ | diffusion | BC-diverse | CFOP |
|---|---|---|---|
| 6  | **0.92** | 0.29 | 1.00 |
| 8  | **0.56** | 0.10 | 1.00 |
| 12 | **0.15** | 0.04 | 1.00 |
| 15 | **0.07** | 0.00 | 1.00 |
| 25 | 0.00 | 0.00 | 1.00 |

学習量による diffusion の向上（depth-8, beam-8）:
`0.34 (20k) → 0.40 → 0.49 → 0.55 → 0.56 (80k)` — 単調だが 60k 以降は鈍化。

### 観察された主要事実
1. **トークン精度は解法能力の指標にならない。** BC は token-acc 0.98 だが解法は
   diffusion の 1/3〜1/5。diffusion は token-acc 0.10〜0.13 で最良。
2. **BC は脆い（共変量シフト）。** 教師軌跡上の状態しか学習しないため、1手でも
   外れると未学習状態に陥り復帰できない。0.98^90 ≈ 0.16。
3. **diffusion は浅い深さに強い**が depth-12 以上で急減衰、depth-25 は 0%。

## 3. 手法レビュー（Fable 5）

> 以下は Fable 5 モデルによる批判的レビューの要点。**現状の数値の多くは、
> 設計・評価上の問題により「測定アーティファクト」を含む**。

### 3.1 【最重要】goal条件付けが一度も学習されていない
全データで goal が identity 固定（`data.py` の `goal_py=_IDENTITY`）。よって
goal 20 トークンと role 埋め込みは全データで定数 → モデルは goal 条件付けを
学習していない。入力の半分が 0 ビットを運び、mean-pool で有効トークンを希釈。
→ 「goal条件付き」は現状未検証・未学習。**hindsight relabeling で goal を多様化
（後述）するか、goal 側を削除して容量を回収すべき。**

### 3.2 目的関数は D3PM ではなく 1ステップ除去
- 変分下界も x0 パラメータ化も posterior 再重み付けもない。ランダムウォークの
  逆カーネルの高分散1サンプル推定にすぎない。
- **逆過程を逆過程として実行していない。** 本来は各ステップで `p(a|x_t,t)` から
  サンプリングすべきところ、`infer.py` は argmax / 累積logprobビーム。意図的に
  多峰な事後分布を argmax するのはカテゴリエラー。
- **`t_max=100` の弊害**: 神の数（20 HTM）を超えると分布は群上でほぼ一様になり、
  ラベルがほぼ無情報。学習サンプルの約75%が混合領域＝ノイズ。loss 下限 2.68
  （≈ ln18=2.89 に対し）と token-acc 0.10 はこの分析と定量的に整合。
  → **depth-25=0% は容量ではなく目的関数の天井。** 学習量・パラメータでは破れない。
- `t` 自体が距離の雑なラベル（`U U'` の相殺、`U U`→`U2` の併合）。

### 3.3 推論の t スケジュールは無根拠な開ループ制御
`t = max(1, scramble_depth - k)` は「1手ごとに必ず1歩前進」を仮定。1手外すと
真の距離は増えるのにスケジュールは減り、条件付けが単調に分布外へドリフト。
さらに `max(1, ·)` クランプで予算の3/4が `t=1`（=「solvedまで1手」）で実行される。

### 3.4 評価ハーネスの欠陥（結論の信頼性を損なう）
- **BC の `max_steps` が教師解より短い**（`scramble_depth*4`、depth-6で24手 vs
  CFOP約90手）。完璧なクローンでも depth-6 で約0% になる。BC vs diffusion 比較は
  この点で不当。
- **train と eval で t の意味が不一致**（BC: 残り手数、eval: `scramble_depth-k`）。
- **ビームの累積logprobは無効な基準**。1ステップ除去器の出力の積は軌跡品質と無関係。
  `U U' U U'` のような自信ある振動を高評価。状態重複排除・直前逆手の禁止もなし。
- **貪欲にサイクル検出なし**。状態が一巡すると残り予算を無駄にループ。
- **ホールドアウト検証なし**（`val_batch` を学習プールから再サンプル）。0.98 は
  訓練精度・部分的に暗記。
- n=100 は CI 広すぎ（depth-12 0.15 vs depth-15 0.07 はほぼ有意差なし）。
- 「深さ」は適用手数であり距離ではない（相殺で距離 < 深さ）。

### 3.5 容量がボトルネックという証拠は弱い
loss 下限が多峰性分析の予測位置にあり、depth-25=0% は目的関数から構造的に導かれる。
**まず per-t に層別した loss/acc を記録**すれば容量 vs 目的関数を切り分けられる
（medium 学習はこの診断を兼ねる）。

## 4. 今後のロードマップ（インパクト ÷ 労力で優先）

| # | 改善 | インパクト | 労力 |
|---|---|---|---|
| 1 | **評価ハーネス修正**（全結論の前提）: モデル別 t 意味、BC の max_steps≥教師長、ビーム重複排除＋逆手禁止、貪欲サイクル検出、n≥500、ホールドアウト検証、ckpt に config 保存 | 高（全数値の信頼回復） | 低 |
| 2 | **t 分布の上限/再形成**: `t ≤ 約26` にする or 小 t を強く重み付け。正準スクランブルで t≈距離。学習計算の約75%（ノイズ）を回収。同じ small で再学習 | 高 | 低 |
| 3 | **cost-to-go（価値）ヘッド**: 同じウォークから距離を回帰、理想は DeepCubeA 流の価値反復 `J(x)=1+min_a J(x·a)`。ビーム/A* の基準に。開ループ t と無効ビーム基準を同時に解消 | 非常に高 | 中 |
| 4 | **hindsight relabeling**: ウォーク区間 `(x_i…x_j)` で goal=x_i, current=x_j, t=j−i。同じ生成器で t 倍のサンプル、かつ **goal 側を初めて学習**（§3.1解消）、小 t 密データも得られる | 高 | 低〜中 |
| 5 | **推論はサンプリング**（argmax でなく温度1、best-of-k）。学習した逆カーネルに整合し、決定的サイクルを破る | 中 | 些少 |
| 6 | **BC は DAgger**: 生徒をロールアウト→訪問した分布外状態で CFOP に問い合わせプールに追加。greedy 復号なら `--diverse-pool` は外す | 中（BC系のみ） | 中 |
| 7 | **モデル拡大（medium 等）**: §3.5 の per-t 診断で小 t に容量律速が見えてから。現時点では時期尚早 | 不明 | 中 |

### 最優先 3 つ
1. **ハーネス修正**（max_steps・t意味・ビーム重複排除・ホールドアウト）— 現状の
   BC比較と BC 数値はアーティファクトを含む。
2. **t を約26に制限して再学習** — 現在の学習信号の大半は証明可能にノイズ。最小の変更で最大の効果。
3. **価値ヘッド + 探索（DeepCubeA流）** — depth-12+ の壁は容量でなく目的関数の天井。
   cost-to-go を予測し探索するのが定石で、開ループ t と無効ビーム基準も同時に解決。

## 5. Fable5 改善 #1〜#4 の実装と結果（2026-06）

ロードマップ §4 の #1〜#4 を `feature/harness-fixes` で実装し検証した。

### 5.1 実装
- **#1 ハーネス修正**: モデル別 t 意味（`--t-mode countdown|const|none`）、ビーム重複排除＋
  直前逆手の禁止、貪欲サイクル検出、ckpt に config json を併存保存、ホールドアウト検証
  （学習プールと別シードの固定 2048 バッチ）。
- **#2 t上限**: `--t-cap 26`。神の数超のノイズ領域を学習から除外。
- **#4 hindsight relabeling**: `generate_batch_hindsight` — ウォーク区間 (x_i…x_j) で
  goal=x_i, current=x_j, t=j−i。`--identity-goal-frac 0.5` で半分は goal=identity。
  **goal 側が初めて学習される**（§3.1 の解消）。
- **#3 価値ヘッド + 価値誘導探索**: `solver.py` に `value_head`、`return_value=True` で
  (logits, value) を返す。`train.py` は Huber で t を回帰（`--value-weight`）。`infer.py` に
  `rollout_value_beam`（`--search value-beam`）= ポリシー提案 + 価値ランキングのビーム。

### 5.2 【重要な落とし穴】価値ヘッドの t-echo
価値ヘッドに **入力 t を与えると、価値ヘッドは t をそのまま echo するだけ**になる
（MAE 0.065 と異常に低い。t=8入力→状態によらず全部 ~8.0）。ビーム内の全子ノードが
同一 t を受けるため価値が同一化し、**value-beam が完全に破綻**（depth-8 で 0.01）。
→ `--no-t` で t を一切与えず、価値ヘッドが**状態だけから cost-to-go を学ぶ**
DeepCubeA 流に修正。value_mae は本物の ~2.2（t∈[1,26]）に。これで value-beam が機能。

### 5.3 結果（small 812K, hindsight + no-t + value, n=100）

| depth | greedy | logprob beam(8) | **value-beam(8)** | 旧 diffusion(beam) |
|---|---|---|---|---|
| 8  | 0.51 | 0.85 | **0.93** | 0.56 |
| 12 | —    | 0.35 | **0.56** | 0.15 |
| 15 | —    | 0.17 | **0.31** | 0.07 |
| 25 | —    | 0.00 | **0.01** | 0.00 |

- ホールドアウト val_acc 0.10（旧 diffusion）→ **0.44**。goal 条件付けが初めて機能した効果。
- 同じ 812K で **depth-12 が 3.7倍、depth-15 が 4.4倍**。目的関数の天井が大きく後退。
- **value-beam が全深さで logprob beam を上回る** — #3 が一貫して寄与。
- depth-25 は依然ほぼ 0。25手は神の数に近く、t-cap=26 の学習分布の端でもある。
  ここを破るには容量拡大（§4 #7）か真の価値反復（DeepCubeA の `J(x)=1+min_a J(x·a)`、
  現状は walk 長 t の回帰＝距離の上界に留まる）が必要。

### 5.4 次の一手
1. **真の cost-to-go**: 現状の価値ターゲットは walk 長 t（真の距離の上界、相殺で過大）。
   バッチ内で `J(x)=1+min_a J(x·a)` の価値反復にすると距離が引き締まり depth-15+ が伸びる見込み。
2. **モデル拡大**（medium 4.8M）: §3.5 の per-t 診断と併せ、小 t に容量律速が見えてから。
3. **n≥500 で CI を締める**: 現状 n=100 は depth 間の差の一部が有意でない可能性。
